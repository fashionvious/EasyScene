# 为什么我彻底放弃了普通 AI 助手，全面转向 OpenCode？

这两年我用过无数的 AI 编程工具，但最终，我把所有的主力工作流都交给了 OpenCode。

理由很简单：我受够了被单一平台绑架，更受够了那些在终端里卡顿、笨拙的玩具。如果你跟我一样，是一个追求极致效率的终端极客，那 OpenCode 绝对是你的终极武器。

它跟市面上爆火的 Claude Code 非常像，但在核心体验上，OpenCode 做出了截然不同的硬核选择。今天我们就来聊聊，它凭什么能让我彻底入坑。话不多说，我们直接开始。

## 100% 的真正开源与模型自由

用 OpenCode 的第一大安全感，来自于它 100% 开源且不挑模型（Provider-agnostic）。

现如今的模型迭代速度太快了。今天 Claude 称王，明天可能就是 OpenAI 或者本地开源模型登顶。用那些绑定死一家的工具，意味着你永远处于被动。

但 OpenCode 给出了完美的解法：你可以用官方推荐的 OpenCode Zen，可以丝滑接入 Claude、OpenAI、Gemini，甚至可以跑你本地的模型。这种彻底的自由，让你永远能用上最聪明、性价比最高的 AI 大脑。这种掌控感，是闭源商业软件给不了的。

## 为极客而生的终端体验

OpenCode 的第二大优势，就是快，极致的快。

它的开发团队是一群硬核的 Neovim 爱好者。所以它不仅是一个命令行工具，更拥有极度丝滑的 TUI（终端用户界面）。在终端里穿梭、查看代码、执行任务，肉眼可见的流畅，完全不打断你写代码时的心流。简直就是终端界的巧克力。

而且它的架构极其超前。采用客户端/服务端分离设计，你甚至可以在本机跑服务，用手机远程驱动它。更别提它还内置了 LSP（语言服务器）支持，它是真正“懂”你的代码逻辑，而不是在文本里瞎猜。

## 进退自如的双面 Agent

OpenCode 的第三大绝杀，是它极其聪明的 Agent 调度机制。

敲一下 Tab 键，你就能在两种模式中无缝切换：
默认的 `build` 模式火力全开，拥有完整权限，适合直接上手写代码、改Bug。
而 `plan` 模式则是只读的。当你接手一个完全陌生的代码库，或者只需要做架构分析时，用它探路最合适。在这个模式下，它默认拒绝修改任何文件，执行 Bash 命令前也会主动询问。

一个负责冲锋陷阵，一个负责运筹帷幄。如果你有极其复杂的多步任务，还可以随时召唤 `@general` 这个隐藏帮手。安全与高效，被它平衡到了极致。

## 极速上手，告别折腾

最后，它的安装真的是傻瓜式操作。

喜欢 YOLO 精神？终端里直接一行 `curl -fsSL https://opencode.ai/install | bash`，瞬间搞定。
用 Mac 的兄弟直接 `brew install`，Windows 用户也有 `scoop` 和 `choco` 支持。如果你实在不喜欢命令行，官方甚至提供了打包好的 Desktop 桌面端。

全套方案开源免费，不被平台绑架，还能把终端的极限压榨到极致。如果你也想体验这种真正属于极客的 AI 编程心流，现在就去试试 OpenCode 吧。
